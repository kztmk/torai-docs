# Google シートと Apps Script の設定

Google シートと Google Apps Script は、虎威の自動投稿を実行する土台です。

虎威の画面で投稿を作成すると、投稿データは Google シートへ連携されます。Google Apps Script は、指定した間隔で投稿予定時刻を確認し、投稿時刻になったポストを X へ送信します。

この設定は最初に一度行います。Google アカウント、スプレッドシート、Apps Script、Web アプリ URL を同じ組み合わせで管理してください。

## 作業前に用意するもの

- Google アカウント
- 虎威で使用する Google シート
- Apps Script に貼り付ける虎威用コード
- X Developer Console で設定する Callback URL として使う Web アプリ URL

:::caution
Google Apps Script の Web アプリ URL は、虎威と X Developer Console の両方で使用します。あとで確認できるよう、メモ帳やパスワードマネージャーなど安全な場所に控えてください。
:::

## 1. Google シートを作成する

Google にログインし、Google スプレッドシートを開きます。

「空白のスプレッドシート」を選択し、新しいシートを作成します。

> キャプチャー画像：Google スプレッドシートの新規作成画面

左上のファイル名をクリックし、用途が分かる名前に変更します。

例:

- `torai-x-auto-post`
- `虎威_自動投稿`
- `torai_投稿管理`

> キャプチャー画像：スプレッドシート名を変更した状態

## 2. Apps Script を開く

スプレッドシート上部のメニューから「拡張機能」→「Apps Script」を選択します。

> キャプチャー画像：Google シートの「拡張機能」メニューで Apps Script を選択している状態

Apps Script エディタが別タブで開きます。プロジェクト名も、シートと同じように分かりやすい名前へ変更します。

> キャプチャー画像：Apps Script プロジェクト名を変更する画面

## 3. 初期ファイルを整理する

Apps Script の左側にあるファイル一覧で、最初から用意されている `コード.gs` を削除します。

> キャプチャー画像：Apps Script のファイル一覧で「コード.gs」のメニューを開いた状態

確認ダイアログが表示されたら削除します。

次に、ファイル一覧の「＋」を押し、「スクリプト」を追加します。ファイル名は `code` にします。

> キャプチャー画像：Apps Script で新しいスクリプトファイル `code` を作成した状態

エディタに初期コードが入っている場合は、すべて削除します。

## 4. 虎威用コードを貼り付ける

配布されている虎威用 Apps Script コードを開き、全文をコピーします。

Apps Script の `code` ファイルへ貼り付けます。

> キャプチャー画像：Apps Script エディタに虎威用コードを貼り付けた状態

貼り付けたら、上部の保存ボタンを押します。

:::info
コードを更新した場合も、同じように Apps Script のコードを差し替えて保存します。コード更新後は、必要に応じて Web アプリを再デプロイしてください。
:::

## 5. Web アプリとしてデプロイする

Apps Script 右上の「デプロイ」から「新しいデプロイ」を選択します。

> キャプチャー画像：Apps Script の「デプロイ」メニューで「新しいデプロイ」を選択した状態

表示された画面で、種類の選択から「ウェブアプリ」を選びます。

> キャプチャー画像：新しいデプロイ画面で「ウェブアプリ」を選択している状態

次のように設定します。

| 項目                   | 設定                                              |
| ---------------------- | ------------------------------------------------- |
| 説明                   | 自分で分かる説明を入力します。例: `虎威 自動投稿` |
| 次のユーザーとして実行 | 自分                                              |
| アクセスできるユーザー | 全員                                              |

> キャプチャー画像：ウェブアプリのデプロイ設定を入力した状態

設定できたら「デプロイ」をクリックします。

## 6. Google の権限を許可する

初回デプロイ時は、Google から権限の確認が表示されます。

「アクセスを承認」をクリックし、シートを作成した Google アカウントを選択します。

> キャプチャー画像：Google のアクセス承認画面

「Google が確認していないアプリ」と表示される場合があります。自分で作成した Apps Script を実行するための確認です。

表示された場合は、「詳細」または「Advanced」を開き、プロジェクト名のリンクから続行します。

> キャプチャー画像：「Google が確認していないアプリ」画面で詳細を開いた状態

必要な権限の一覧を確認し、許可します。

> キャプチャー画像：Apps Script の権限許可一覧画面

:::caution
ここで選ぶ Google アカウントは、Google シートと Apps Script を管理するアカウントです。別の Google アカウントで許可すると、シートやドライブへのアクセスがうまく動かない原因になります。
:::

## 7. Web アプリ URL をコピーする

デプロイが完了すると、Web アプリ URL が表示されます。

この URL をコピーして保管します。

> キャプチャー画像：デプロイ完了画面で Web アプリ URL が表示されている状態

この URL は次の設定で使います。

- X Developer Console の Callback URL / Redirect URL
- 虎威の API キー画面での GAS URL 登録

## 8. 動作確認の準備

ここまで完了したら、次に虎威へ GAS URL を登録します。

虎威側で本人確認を行うと、Google シートに本人確認コードが作成されます。そのコードを虎威へ入力して保存すると、虎威アカウントとこの Apps Script が連携されます。

:::caution
一度連携した Apps Script は、別の虎威アカウントへ移転できません。移転したい場合は、シートのデータだけを新しいシートへコピーし、新しい虎威アカウントで Apps Script を作り直してください。
:::

詳しい貼り付け、デプロイ、再読み込み、虎威連携の手順は [Apps Script の貼り付け・デプロイ・虎威連携](./apps-script-deploy-link) を参照してください。

## よくある失敗

### Web アプリ URL をコピーし忘れた

Apps Script の「デプロイ」→「デプロイを管理」から、現在の Web アプリ URL を確認できます。

### デプロイ後にコードを更新した

コードを保存しただけでは、Web アプリへ反映されない場合があります。「デプロイを管理」から新しいバージョンとして再デプロイしてください。

### Google 認証で止まる

作成した Google アカウントと、権限許可に使っている Google アカウントが同じか確認してください。

### 虎威で「別のアカウントに連携されています」と表示される

その Apps Script はすでに別の虎威アカウントに紐づいています。乗っ取り防止のため、別アカウントへの移転はできません。新しい Google シートと Apps Script を作り直してください。
