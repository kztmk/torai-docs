# スレッド投稿

虎威では、複数のポストをリプライでつなぐスレッド投稿を予約できます。

スレッド投稿は、1つ目のポストに対して2つ目以降をリプライとして投稿する形式です。3つ目の投稿は2つ目のリプライとして投稿します。1連の投稿をストーリーとして伝えることができる、また、ポスト表示でもつながって表示されるので目立ちます。

ただし、複数の投稿を連続して行うため、通常ポストより API 利用量が増えます。

## 通常ポストとの違い

| 種類         | 内容                                 |
| ------------ | ------------------------------------ |
| 通常ポスト   | 1件のポストとして投稿します。        |
| スレッド投稿 | 複数のポストをリプライでつなげます。 |
| 引用ポスト   | 既存のポストを引用して投稿します。   |

### 引用ポスト

　引用ポストは引用先がリンクと見なされ通常より高額なAPI使用料金がかかります。

![引用ポスト](/img/torai/post/threads10.jpg)

### スレッド投稿のメリット

スレッド投稿のメリットは、

- タイムラインでも一部省略されますがスレッドとして表示され、目立つ
- ストーリー性を持つので興味を持たれやすい
- たとえ自分でもリプライがあると評価が上がる

ため、エンゲージメントに貢献します。

## 1. スレッド用のポストを作成する

まず、通常ポストでスレッドに含める各ポストを作成します。

![スレッド投稿用を普通に作成](/img/torai/post/threads1.jpg)

1つ目が親ポスト、2つ目以降がリプライとして投稿されます。順番が重要です。

## 2. スレッド投稿作成画面を開く

投稿一覧でスレッドに含める投稿を選択し、スレッド作成ボタンをクリックします。

![スレッドに含める投稿を選択](/img/torai/post/threads2.jpg)

## 3. スレッドに含めるポストを選択する

スレッドから除外する投稿を「ゴミ箱」アイコンで除外します。

![スレッドメンバー決定](/img/torai/post/threads3.jpg)

スレッド投稿のメンバーが決まれば「作成」ボタンをクリックします。

スレッドの親ポスト、子ポストが表示されます。

![スレッド投稿表示](/img/torai/post/threads4.jpg)

## 4. 投稿予定時刻を設定する

虎威は、先頭ポストを投稿したあと、続くポストをリプライとして投稿します。

スレッドの投稿開始日時を設定します。個別に投稿時刻を設定しても良いですし、一括で設定することもできます。

スレッド投稿の親ポストを選択するとスレッド投稿に含まれるポストが選択されます。

![スレッド投稿選択](/img/torai/post/threads5.jpg)

上部にある「一括投稿時刻設定」ボタンをクリックします。

## 5. 投稿日時を設定し保存する

- 開始日、終了日
- 開始時刻、終了時刻
- 投稿間隔

を設定し「作成」ボタンをクリックします。

![一括投稿時刻設定](/img/torai/post/threads6.jpg)

## 6. 投稿後に確認する

自動投稿後、X 上でスレッドが正しい順序でつながっているか確認します。

![スレッド投稿完了](/img/torai/post/threads7.jpg)

## 注意点

- スレッド内のポスト数だけ API 投稿が発生します。
- 途中で投稿に失敗すると、スレッドが途中で止まる場合があります。
- 画像や動画を含むスレッドは、投稿に時間がかかる場合があります。
- 大量のスレッド投稿をまとめて予約する前に、X Developer Console の Usage を確認してください。
