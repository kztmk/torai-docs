# スレッド投稿

虎威では、複数のポストをリプライでつなぐスレッド投稿を予約できます。

スレッド投稿は、1つ目のポストに対して2つ目以降をリプライとして投稿する形式です。複数の投稿を連続して行うため、通常ポストより API 利用量が増えます。

## 通常ポストとの違い

| 種類 | 内容 |
| --- | --- |
| 通常ポスト | 1件のポストとして投稿します。 |
| スレッド投稿 | 複数のポストをリプライでつなげます。 |
| 引用ポスト | 既存のポストを引用して投稿します。 |

## 1. スレッド用のポストを作成する

まず、スレッドに含める各ポストを作成します。

> キャプチャー画像：スレッドに使う複数のポストを作成した状態

1つ目が親ポスト、2つ目以降がリプライとして投稿されます。順番が重要です。

## 2. スレッド投稿作成画面を開く

投稿一覧またはスレッド作成メニューから、スレッド投稿作成画面を開きます。

> キャプチャー画像：スレッド投稿作成画面を開いた状態

## 3. スレッドに含めるポストを選択する

スレッドに含めたいポストを選択します。

> キャプチャー画像：スレッドに含めるポストを選択している状態

選択後、投稿順を確認します。

> キャプチャー画像：スレッド投稿の順番を確認している状態

## 4. 投稿予定時刻を設定する

スレッドの開始予定時刻を設定します。

> キャプチャー画像：スレッド投稿の開始予定時刻を設定した状態

虎威は、先頭ポストを投稿したあと、続くポストをリプライとして投稿します。

## 5. 保存して確認する

設定内容を保存し、スレッド投稿が一覧に表示されていることを確認します。

> キャプチャー画像：スレッド投稿が一覧に表示された状態

## 6. 投稿後に確認する

自動投稿後、X 上でスレッドが正しい順序でつながっているか確認します。

> キャプチャー画像：X 上でスレッド投稿がつながっている状態

## 注意点

- スレッド内のポスト数だけ API 投稿が発生します。
- 途中で投稿に失敗すると、スレッドが途中で止まる場合があります。
- 画像や動画を含むスレッドは、投稿に時間がかかる場合があります。
- 大量のスレッド投稿をまとめて予約する前に、X Developer Console の Usage を確認してください。
